CinemaFlix API - Documentation

Overview

This is an API for a movie database app called CinemaFlix in which users can search for and save information about movies. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.


Request-Response Guidelines for URL Endpoints

Request URL HTTP Method Request Body Data Format Response Body Data Format
Get a list of all movies to a user /movies GET None A JSON object holding data about all the movies
Get data about a single movie by title /movies/[title] GET None A JSON object holding data about a single movie.
Example: 
                    
  {
    "Title": "The Graduate",
    "ReleaseYear": "1967",
    "Description": "The Graduate is a Comedy film...",
    "Genre": [
      "Name":"Comedy",
      "Description":"Comeady film genere is..."
    ],
    "Director":
      "Name": "Mike Nichols",
      "BirthYear": "1980"
      "Bio": "Mike Nichols is an American director, producer...."
    "Actors": [
      "Anne Bancroft",
      "Dustin Hoffman",
      "Katharine Ross",
      "William Daniels"
    ],
    "imageUrl": "http://ia.media-imdb.com/images/M/MV5BM.jpg",
    "Featured": "True"
  }
Get a list of all genres /genres GET None A JSON object holding data about all the genres
Get a data about a single genre by name /genres/[name] GET None A JSON object holding data about a single genre.
Example: 
                    
  {
      "Name": "Action",
      "Description": "Action film is a film genre in which the protagonist
      or protagonists are thrust into a series of events that typically
      include violence, extended fighting, physical feats, rescues and
      frantic chases."
  }
Get a list of all directors /directors GET None A JSON object holding data about all the directors
Get a data about a single director by name /directors/[name] GET None A JSON object holding data about a single directors.
Example: 
                    
  {
      "Name": "Mike Nichols",
      "Bio": "Mike Nichols is an American director, producer....",
      "BirthYear": "1980"
  }
Allow new users to register /users POST A JSON object holding data about the new user to be added.
Example: 
                    
  {
    "Username": "Brad Tomson",
    "password": "son1990",
    "Email": "tomsonb@mail.com",
    "Birthday": "01.10.1990",
  }
A JSON object holding data about the new added user, including the new ID: Example 
                    
  {
    "Username": "Brad Tomson",
    "password": "son1990",
    "Email": "tomsonb@mail.com",
    "Birthday": "01.10.1990",
    "FavoriteMovies": [],
    "id": "txsddf0222-xr2t-0000",
  }
Allow users to update their user info /users/[username] PUT None A confirmation message that a new username was succesfully assigned.
Allow users to add a movie to their favorite list /users/[username]/movies/[MovieId] POST None A confirmation message that the movie is succesfully added to their favorite list.
Allow users to remove a movie from their favorite list /users/[username]/movies/[MovieId] DELETE None A confirmation message that the movie is succesfully removed from their favorite list.
Allow exsisting users to deregister /users/[username] DELETE None A confirmation message that the user is succesfully deregistered.
